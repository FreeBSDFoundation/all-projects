# Project Brief

## Software Bill of Materials (SBOM) Toolchain

| Project Name | Software Bill of Materials (SBOM) Toolchain \- 2026 |
| :---- | :---- |
| Funding Body | FreeBSD Foundation |
| Funding Status | Approved Jan 01, 2026 |
| Project Sponsor | [Deb Goodkin](mailto:deb@freebsdfoundation.org) |
| Project Manager | [Alice Sowerby](mailto:alice@freebsdfoundation.org)  |
| Scheduling | Jan 2026 \- April 2026 |
| Objectives | *Add SBOM information files (Likely a subset of the entire FreeBSD repository, with portions to be determined). Identify files with missing licenses and add where possible Update pkgconf for complex licensing schemes Collaborate with upstream projects to add SBOM metadata where possible* |
| Organization goals | In 2025, The Sovereign Tech Agency commissioned the FreeBSD Foundation to manage work that included Work Package E, which was focused on SBOM improvements. More information about the STA and these projects can be found here:  [STA Work Package E](https://github.com/FreeBSDFoundation/all-projects/tree/main/Infrastructure%20Modernization%20\(STA%20commissioned\)/E_Software%20Bill%20of%20Materials%20\(Work%20Package%20E\)) Going into 2026, the FreeBSD Foundation is expanding its work to ensure that the FreeBSD Project is prepared for new regulations on the software industry (the [EU’s Cyber Resilience Act](https://orcwg.org/cra/) in 2027).  |
| Outputs | Native support for SBOM generation in the base system. Better visibility of dependencies through pkgconf.  Simplified build system for third-party and downstream software. Streamlined maintenance and updates through vendor import workflow. Improved reproducibility through consistent metadata handling. |
